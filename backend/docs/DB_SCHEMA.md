# Database Schema — News App (Firebase Firestore)

## Overview

Firebase Firestore is used as the primary database. Firebase Cloud Storage handles all media files (thumbnails) under the `media/articles/` folder.

---

## Collections

### `articles`

Each document represents a news article.

| Field         | Type        | Description |
|---------------|-------------|-------------|
| `id`          | `string`    | Document ID (auto-generated by Firestore) |
| `title`       | `string`    | Headline of the article |
| `description` | `string`    | Short summary / subtitle |
| `content`     | `string`    | Full article body text |
| `author`      | `string`    | Author name |
| `sourceName`  | `string`    | News outlet name (e.g. "BBC News") |
| `sourceId`    | `string`    | News outlet identifier slug |
| `category`    | `string`    | One of: `general`, `technology`, `sports`, `business`, `entertainment`, `health`, `science` |
| `thumbnailURL`| `string`    | **Reference to Firebase Cloud Storage** — path `media/articles/{articleId}.jpg` (download URL) |
| `publishedAt` | `timestamp` | Publication datetime |
| `url`         | `string`    | Link to the original article |
| `isBreaking`  | `boolean`   | Whether the article is a breaking-news item |
| `tags`        | `array<string>` | Searchable keywords/tags |
| `views`       | `number`    | View counter |

**Example document:**

```json
{
  "id": "abc123",
  "title": "Flutter 4.0 Released with Major Performance Improvements",
  "description": "Google's UI toolkit gets a massive update targeting sub-16ms frame rendering.",
  "content": "Google today announced...",
  "author": "Jane Doe",
  "sourceName": "TechCrunch",
  "sourceId": "techcrunch",
  "category": "technology",
  "thumbnailURL": "https://firebasestorage.googleapis.com/.../media%2Farticles%2Fabc123.jpg",
  "publishedAt": "2026-03-11T09:00:00Z",
  "url": "https://techcrunch.com/...",
  "isBreaking": false,
  "tags": ["flutter", "google", "mobile", "performance"],
  "views": 1240
}
```

---

### `categories`

Lookup collection — one document per category.

| Field    | Type     | Description |
|----------|----------|-------------|
| `id`     | `string` | e.g. `technology` |
| `label`  | `string` | Display name, e.g. `Technology` |
| `iconUrl`| `string` | Optional icon from Storage |

---

## Firebase Cloud Storage

All article thumbnails are stored under:

```
media/
  articles/
    {articleId}.jpg
```

The `thumbnailURL` field in Firestore holds the **full HTTPS download URL** returned by Firebase Storage after upload, so clients can load the image directly without extra Storage calls.

---

## Indexes

- `articles` collection: composite index on `(category ASC, publishedAt DESC)` — for category-filtered feeds sorted by date.
- `articles` collection: composite index on `(isBreaking DESC, publishedAt DESC)` — for breaking-news queries.

---

## Design Decisions

1. **Flat structure over sub-collections** — articles belong to one category; embedding category as a string field is simpler and covers all current query patterns.
2. **Thumbnail as download URL** — storing the full download URL avoids an extra Storage SDK call on every list item render, improving feed performance.
3. **`tags` array** — enables Firestore `array-contains` queries for tag-based filtering without a separate index collection.
4. **`views` counter** — stored directly on the document; for high-traffic apps, consider using distributed counters or Firebase Functions to increment atomically.
